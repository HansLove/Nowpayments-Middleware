# Dispersion Guide

On-chain token dispersion allows you to route NowPayments payouts through a controlled intermediate wallet and then automatically send tokens to the final user wallet. This solves the NowPayments limitation that requires pre-registered withdrawal addresses.

## How It Works

The flow has three stages:

1. **Payout creation** (`createPayoutWithDispersion`): Send payout to your controlled wallet. Register the final user wallet as a dispersion target.
2. **Payout webhook** (`payoutWebhook`): When NowPayments confirms the payout as `finished`, the middleware automatically triggers dispersion.
3. **Dispersion execution** (`DispersionOrchestrator`): The provider sends tokens from the controlled wallet to the final user wallet on-chain.

```
NowPayments -> Controlled Wallet -> (on-chain transfer) -> User Wallet
```

## Installation

Install the peer dependencies for the networks you need:

```bash
# For Polygon (and any EVM network)
pnpm add ethers@^6

# For Tron
pnpm add tronweb@^6
```

## Setup

### 1. Configure providers

```typescript
import {
  NowPaymentsMiddleware,
  PolygonDispersionProvider,
  TronDispersionProvider,
} from '@taloon/nowpayments-middleware';

NowPaymentsMiddleware.configure({
  apiKey: process.env.NOWPAYMENTS_API_KEY!,
  email: process.env.NOWPAYMENTS_EMAIL,
  password: process.env.NOWPAYMENTS_PASSWORD,
  twoFactorSecretKey: process.env.NOWPAYMENTS_2FA_SECRET,
  dispersion: {
    providers: [
      new PolygonDispersionProvider({
        network: 'polygon',
        rpcUrl: process.env.POLYGON_RPC_URL!,
        controlledAddress: process.env.CONTROLLED_POLYGON_ADDRESS!,
        privateKey: process.env.CONTROLLED_POLYGON_PRIVATE_KEY!,
        tokenContractAddress: '0xc2132D05D31c914a87C6611C10748AEb04B58e8F',
        tokenDecimals: 6,
      }),
      new TronDispersionProvider({
        network: 'tron',
        fullNode: 'https://api.trongrid.io',
        controlledAddress: process.env.CONTROLLED_TRON_ADDRESS!,
        privateKey: process.env.CONTROLLED_TRON_PRIVATE_KEY!,
        tokenContractAddress: 'TR7NHqjeKQxGTCi8q8ZY4pL8otSzgjLj6t',
        tokenDecimals: 6,
      }),
    ],
    callbacks: {
      onStatusChange: async (withdrawalId, _prev, status, ctx) => {
        await db.updateWithdrawal(withdrawalId, {
          dispersion_status: status,
          ...(ctx.txHash && { tx_hash: ctx.txHash }),
          ...(ctx.error && { error: ctx.error }),
        });
      },
      onDispersionFailed: async (withdrawalId, error, network) => {
        await alerting.notify(`Dispersion failed ${withdrawalId} on ${network}: ${error}`);
      },
      onGasCritical: async (network, balance, threshold) => {
        await alerting.notifyUrgent(`Top up ${network} gas: ${balance} ${threshold.currency}`);
      },
    },
    gasThresholds: {
      polygon: { minimum: '0.1', critical: '0.05', currency: 'MATIC' },
      tron: { minimum: '10', critical: '5', currency: 'TRX' },
    },
  },
});
```

### 2. Create payout endpoint

Use `createPayoutWithDispersion` instead of `createPayout`. Add `dispersion_*` fields to each withdrawal - they are stripped before the request reaches the NowPayments API.

```typescript
app.post('/withdrawals',
  NowPaymentsMiddleware.createPayoutWithDispersion({
    mapRequest: (req) => ({
      withdrawals: [{
        // Sent to NowPayments (your controlled wallet)
        address: getControlledAddress(req.body.network),
        currency: req.body.currency,
        amount: req.body.amount,
        ipn_callback_url: `${BASE_URL}/webhook/payout`,
        // Dispersion metadata (stripped before API call)
        dispersion_network: req.body.network,          // 'polygon' | 'tron'
        dispersion_final_address: req.body.userWallet, // end user's wallet
        dispersion_amount: req.body.amount.toString(),
      }],
    }),
  }),
  (req, res) => {
    res.json({ success: true, data: res.locals.nowPaymentsResponse });
  }
);
```

### 3. Configure payout webhook

The `onFinished` callback runs before dispersion. Use it for your own business logic (mark payout complete in DB, etc.) before the dispersion state transitions begin.

```typescript
app.post('/webhook/payout',
  NowPaymentsMiddleware.payoutWebhook({
    onFinished: async (payload) => {
      // Runs BEFORE dispersion starts
      await db.markPayoutFinished(payload.id, payload.hash);
    },
    onFailed: async (payload) => {
      await db.markPayoutFailed(payload.id, payload.error);
    },
  })
);
```

## Lifecycle and Callbacks

Dispersion transitions through four statuses. The `onStatusChange` callback fires on every transition and is the primary hook for persistence.

```
null -> PENDING -> PROCESSING -> SENT
                             -> FAILED (sendToken error or gas critical)
     -> FAILED (no provider found)
```

### Callback execution order (for FINISHED webhook)

1. `onFinished` (user business logic)
2. `onStatusChange(id, null, 'pending', { network })`
3. `onDispersionStart(id, target, network)`
4. Gas check (if `gasThresholds` configured)
   - `onGasLow` if balance < minimum
   - `onGasCritical` + `FAILED` if balance < critical
5. `onStatusChange(id, 'pending', 'processing', { network })`
6. `provider.estimateGasFee()`
7. `provider.sendToken()`
8. On success: `onStatusChange(id, 'processing', 'sent', { txHash, estimatedGasFee })` + `onDispersionSent`
9. On failure: `onStatusChange(id, 'processing', 'failed', { error })` + `onDispersionFailed`

### Callback error isolation

Errors thrown inside any dispersion callback are silently caught and do not affect the webhook response. The webhook always returns `200 OK`.

## Gas Management

Configure thresholds per network to receive alerts before gas runs out:

```typescript
gasThresholds: {
  polygon: {
    minimum: '0.1',  // isLow=true at this level (warn but proceed)
    critical: '0.05', // isCritical=true at this level (block dispersion)
    currency: 'MATIC',
  },
  tron: {
    minimum: '10',
    critical: '5',
    currency: 'TRX',
  },
},
```

When `isCritical=true`, dispersion is blocked and `onDispersionFailed` fires immediately.

## Custom Provider

Extend `BaseDispersionProvider` to support other EVM networks or custom chains:

```typescript
import {
  BaseDispersionProvider,
  EvmProviderConfig,
} from '@taloon/nowpayments-middleware';

export class BscDispersionProvider extends BaseDispersionProvider<EvmProviderConfig> {
  // Same implementation as PolygonDispersionProvider but pointed to BSC RPC
  async connect(): Promise<void> {
    const ethers = await import('ethers');
    this.ethersProvider = new ethers.JsonRpcProvider(this.config.rpcUrl);
    // ...
    this.connected = true;
  }

  // implement remaining abstract methods
}
```

## Important Caveats

### In-memory target store

`DispersionTargetStore` is in-memory. If the Node.js process restarts between payout creation and the `onFinished` webhook, the dispersion target is lost. The dispersion will not trigger.

**Mitigation**: Implement `onDispersionFailed` to detect this and alert for manual retry. Alternatively, persist targets in your DB on payout creation and restore them on process start.

### Index-based mapping

`createPayoutWithDispersion` maps `response.withdrawals[i]` to `request.withdrawals[i]` by position. NowPayments preserves withdrawal order, but be aware of this assumption when batching multiple withdrawals.

### TOTP auto-verification

When `twoFactorSecretKey` is configured, the payout is automatically verified after creation using a TOTP code. This is required for payouts to be processed by NowPayments. Without it, the payout will remain in `waiting` status.

### Tron energy estimation

The Tron provider estimates gas fees using the TronGrid energy API. Energy cost varies significantly depending on whether the recipient already has USDT (TRC-20) in their wallet (~65k energy) or not (~130k+ energy). The conservative fallback is `15 TRX`.

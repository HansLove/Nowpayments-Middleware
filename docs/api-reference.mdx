# API Reference

Complete API reference for NowPayments middleware functions and types.

## Middleware Functions

### `createPayment(options)`

Creates a payment middleware for standard payments.

```typescript
interface CreatePaymentMiddlewareOptions {
  mapRequest: (req: Request, res: Response) => CreatePaymentRequest;
  transformResponse?: (response: CreatePaymentResponse) => unknown;
  onError?: ErrorHandler;
}

// Usage
app.post('/payment',
  NowPaymentsMiddleware.createPayment({
    mapRequest: (req, res) => ({
      price_amount: req.body.amount,
      price_currency: req.body.currency,
      pay_currency: req.body.cryptoCurrency,
      order_id: req.body.orderId,
      customer_email: req.body.email,
      ipn_callback_url: 'https://your-domain.com/webhook',
    }),
    transformResponse: (response) => ({
      paymentId: response.payment_id,
      status: response.payment_status,
      payAddress: response.pay_address,
    }),
  }),
  (req, res) => {
    const payment = res.locals.nowPaymentsResponse;
    res.json({ payment });
  }
);
```

### `createInvoicePayment(options)`

Creates a composite middleware that internally creates an invoice and then creates a payment from it. This is ideal for fixed-amount payments where you want to accept a specific cryptocurrency without redirecting the user.

```typescript
interface CreateInvoicePaymentMiddlewareOptions {
  mapRequest: (req: Request, res: Response) => CreateInvoicePaymentRequest;
  transformResponse?: (response: CreatePaymentByInvoiceResponse) => unknown;
  onError?: ErrorHandler;
}

interface CreateInvoicePaymentRequest {
  // Invoice fields
  price_amount: number;
  price_currency: string;
  order_id?: string;
  order_description?: string;
  ipn_callback_url?: string;
  success_url?: string;
  cancel_url?: string;
  // Payment field
  pay_currency: string;
  customer_email?: string;
  payout_address?: string;
  payout_extra_id?: string;
  payout_currency?: string;
}

// Usage
app.post('/fixed-payment',
  NowPaymentsMiddleware.createInvoicePayment({
    mapRequest: (req, res) => ({
      price_amount: req.body.amount,
      price_currency: 'usd',
      order_id: req.body.orderId,
      ipn_callback_url: 'https://your-domain.com/webhook',
      pay_currency: 'usdtmatic', // USDT on Polygon
    }),
    transformResponse: (response) => ({
      paymentId: response.payment_id,
      payAddress: response.pay_address,
      payAmount: response.pay_amount,
      status: response.payment_status,
    }),
  }),
  (req, res) => {
    const payment = res.locals.nowPaymentsResponse;
    res.json({ payment });
  }
);
```

**When to use:**

- You want a fixed fiat amount converted to crypto
- You don't have a pre-existing invoice ID
- You want a seamless flow without redirecting to NowPayments

### `createPayoutWithDispersion(options)`

Creates a payout middleware that also registers dispersion targets. When the payout webhook receives `onFinished`, the middleware automatically triggers on-chain token dispersion to the final user wallet.

Requires `dispersion` to be configured in `NowPaymentsMiddleware.configure()`. See the [Dispersion Guide](./dispersion-guide.mdx) for full setup.

```typescript
interface CreatePayoutWithDispersionMiddlewareOptions {
  mapRequest: (req: Request, res: Response) => CreatePayoutWithDispersionRequest;
  transformResponse?: (response: CreatePayoutResponse) => unknown;
  onError?: ErrorHandler;
}

interface CreatePayoutWithDispersionRequest {
  withdrawals: PayoutWithdrawalWithDispersion[];
  ipn_callback_url?: string;
  payout_description?: string;
}

interface PayoutWithdrawalWithDispersion extends PayoutWithdrawal {
  dispersion_network: string;          // Required: 'polygon' | 'tron' | custom
  dispersion_final_address: string;    // Required: end user's on-chain wallet
  dispersion_amount?: string;          // Optional: overrides w.amount for dispersion
  dispersion_token?: string;           // Optional: overrides w.currency for dispersion
}

// Usage
app.post('/withdrawals',
  NowPaymentsMiddleware.createPayoutWithDispersion({
    mapRequest: (req) => ({
      withdrawals: [{
        address: getControlledWalletAddress(req.body.network),
        currency: req.body.currency,
        amount: req.body.amount,
        ipn_callback_url: 'https://your-domain.com/webhook/payout',
        // Dispersion metadata (stripped before sending to NowPayments)
        dispersion_network: req.body.network,
        dispersion_final_address: req.body.userWallet,
        dispersion_amount: req.body.amount.toString(),
      }],
    }),
  }),
  (req, res) => {
    const payout = res.locals.nowPaymentsResponse;
    res.json({ success: true, payout });
  }
);
```

**How it works:**

1. Strips `dispersion_*` fields and sends the clean request to NowPayments API
2. Maps each `response.withdrawals[i]` to `request.withdrawals[i]` by index
3. Registers `DispersionTarget` entries in `DispersionTargetStore`
4. When `payoutWebhook` receives `onFinished`, dispersion triggers automatically

## Request Mapping with res.locals

The `mapRequest` function receives both `req` and `res` parameters, allowing you to access data added by previous middleware through `res.locals`. This is useful when you need to include contextual information (user ID, authentication data, etc.) in the payment request.

### Example: Using Authentication Data

```typescript
// Middleware that adds user info to res.locals
app.use((req, res, next) => {
  res.locals.userId = req.headers['x-user-id'];
  res.locals.accountId = req.headers['x-account-id'];
  next();
});

// Now you can access this data in mapRequest
app.post('/create-payment',
  NowPaymentsMiddleware.createPayment({
    mapRequest: (req, res) => ({
      price_amount: req.body.amount,
      price_currency: req.body.currency,
      pay_currency: req.body.cryptoCurrency,
      order_id: `order_${res.locals.userId}_${Date.now()}`,
      customer_email: req.body.email,
      // Include account-specific webhook URL
      ipn_callback_url: `https://your-domain.com/webhook/payment/${res.locals.accountId}`,
    }),
    transformResponse: (response) => ({
      paymentId: response.payment_id,
      status: response.payment_status,
      payAddress: response.pay_address,
      amount: response.pay_amount,
      currency: response.pay_currency,
    }),
  }),
  (req, res) => {
    const payment = res.locals.nowPaymentsResponse;
    res.json({ success: true, payment });
  }
);
```

## Error Handling Types

### `ErrorHandler`

Type for custom error handling callbacks.

```typescript
type ErrorHandler = (
  error: unknown,
  req: express.Request,
  res: express.Response,
  next: express.NextFunction
) => void | Promise<void>;
```

**Parameters:**

- `error`: The error that occurred (can be NowPaymentsError or any other error type)
- `req`: Express request object
- `res`: Express response object
- `next`: Express next function to pass error to the next middleware

**Usage:**

Per-middleware error handler:

```typescript
app.post('/payment',
  NowPaymentsMiddleware.createPayment({
    mapRequest: (req, res) => ({ /* ... */ }),
    onError: async (error, req, res, next) => {
      console.error('Payment error:', error);

      if (error instanceof NowPaymentsApiError) {
        return res.status(503).json({
          error: 'Payment service unavailable'
        });
      }

      res.status(500).json({ error: 'Payment failed' });
    },
  })
);
```

Global error handler:

```typescript
NowPaymentsMiddleware.configure({
  apiKey: 'your-api-key',
  onError: async (error, req, res, next) => {
    // Handle all NowPayments errors globally
    if (error instanceof NowPaymentsError) {
      return res.status(error.statusCode || 500).json({
        error: error.message
      });
    }
    next(error);
  },
});
```

## Configuration

```typescript
interface NowPaymentsConfig {
  apiKey: string;                    // Required: Your NowPayments API key
  email?: string;                    // Optional: Email for authentication (required for payouts)
  password?: string;                 // Optional: Password for authentication (required for payouts)
  twoFactorSecretKey?: string;       // Optional: TOTP secret for automatic payout verification
  baseURL?: string;                  // Optional: API base URL (default: https://api.nowpayments.io/v1)
  errorHandling?: 'next' | 'direct'; // Optional: Legacy error handling mode (default: 'next')
  onError?: ErrorHandler;            // Optional: Global error handler (takes priority over errorHandling)
  dispersion?: DispersionConfig;     // Optional: Dispersion module configuration
}

// Configuration method
NowPaymentsMiddleware.configure(config: NowPaymentsConfig): void
```

**Error Handling Priority:**

1. Per-middleware `onError` handler (highest priority)
2. Global `onError` in configuration
3. Legacy `errorHandling` mode ('next' | 'direct')

### `DispersionConfig`

```typescript
interface DispersionConfig {
  providers: DispersionProvider[];
  callbacks?: DispersionCallbacks;
  gasThresholds?: Record<string, GasThreshold>;
}

interface GasThreshold {
  minimum: string;  // Warn level (isLow=true, canProceed=true)
  critical: string; // Block level (isCritical=true, canProceed=false)
  currency: string; // e.g. 'MATIC', 'TRX'
}

interface DispersionCallbacks {
  // Primary hook: called on every status transition
  onStatusChange?: (
    withdrawalId: string,
    previousStatus: DispersionStatus | null,
    currentStatus: DispersionStatus,
    context: { network: string; txHash?: string; estimatedGasFee?: string; error?: string }
  ) => void | Promise<void>;

  // Convenience hooks
  onDispersionStart?: (withdrawalId: string, target: DispersionTarget, network: string) => void | Promise<void>;
  onDispersionSent?: (withdrawalId: string, txHash: string, network: string) => void | Promise<void>;
  onDispersionFailed?: (withdrawalId: string, error: string, network: string) => void | Promise<void>;
  onGasLow?: (network: string, currentBalance: string, threshold: GasThreshold) => void | Promise<void>;
  onGasCritical?: (network: string, currentBalance: string, threshold: GasThreshold) => void | Promise<void>;
}
```

### `DispersionOrchestrator`

Singleton that manages provider connections and coordinates dispersion execution.

```typescript
import { DispersionOrchestrator } from '@taloon/nowpayments-middleware';

// Initialized automatically when dispersion is configured via NowPaymentsMiddleware.configure()
// Manual use for advanced scenarios:
await DispersionOrchestrator.initialize(dispersionConfig);
const result = await DispersionOrchestrator.disperse(target);
await DispersionOrchestrator.shutdown();
```

### `DispersionTargetStore`

In-memory singleton that bridges payout creation with webhook-triggered dispersion.

```typescript
import { DispersionTargetStore } from '@taloon/nowpayments-middleware';

// Managed automatically by createPayoutWithDispersion
// Manual use for custom flows:
DispersionTargetStore.register(target);
DispersionTargetStore.registerBatch(targets);
const target = DispersionTargetStore.get(withdrawalId);
DispersionTargetStore.remove(withdrawalId);
```

**Caveat:** In-memory only. If the process restarts between payout creation and webhook delivery, registered targets are lost. Use `onDispersionFailed` callbacks to handle this case.
